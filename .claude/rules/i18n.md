# i18n Conventions

- All user-facing strings through i18n. Never hardcode display text.
- Web: `useTranslations()` from next-intl. Mobile: `useTranslation()` from react-i18next.
- Enum values sent to the API stay in English (`'Day'`, `'Monday'`); use `{value, label}` objects for selects where `label` calls the translator.
- i18n keys for frequency units: `habits.form.unitDay`, `habits.form.unitWeek`, etc.
- i18n keys for day names: `dates.days.*` (full) and `dates.daysShort.*` (abbreviated).
- **pt-BR accents:** Always use proper diacritical marks in Portuguese strings (accents, tildes, cedillas). Never write unaccented Portuguese.
- Shared locale files in `packages/shared/src/i18n/`. Both apps import from there.
